# NTU_COMP30271_CW_RobotSim Execution Instructions

---------------------------------------------

## 0. Dependencies
sudo apt-get update
sudo apt install -y ros-humble-navigation2 ros-humble-nav2-bringup

---------------------------------------------

## 1. Build and Source
cd /home/ntu-user/ros2_ws
colcon build --symlink-install
source install/setup.bash

---------------------------------------------

## 2. Run Unified Launch File (single command launches everything)
cd /home/ntu-user/ros2_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch ntu_robotsim cw_complete.launch.py

ros2 launch ntu_robotsim yolo_unified.launch.py (non-proper logger)
---------------------------------------------

## 3. Object Detection Outputs

### 3.1 Live object counts (updated in terminal and saved to file)
Object counts are printed to the new terminal every 1 seconds.
They are also saved to: /home/ntu-user/ros2_ws/src/NTU_COMP30271_CW_RobotSim/detection_logs

### 3.2 Run YOLO Live Counter (optional, separate terminal)
cd /home/ntu-user/ros2_ws/src/NTU_COMP30271_CW_RobotSim/ntu_robotsim/launch && python3 detection_printer.py

### 3.3 Run YOLO Logger (optional, separate terminal)
cd /home/ntu-user/ros2_ws/src/NTU_COMP30271_CW_RobotSim/ntu_robotsim/launch && python3 detection_logger.py

----------------------------------------------

## 4. Landmark detection
python3 /home/ntu-user/ros2_ws/src/NTU_COMP30271_CW_RobotSim/ntu_robotsim/launch/landmark_csv_logger.py

This single command starts all components in sequence:
  - Gazebo simulation (cwmaze world)          [0s]
  - Robot spawn (Atlas jetbot)                [3s]
  - Odometry-to-TF broadcaster               [5s]
  - OctoMap 3D mapping server                 [6s]
  - RViz2 visualisation                       [5s]
  - Nav2 navigation stack                     [8s]
  - Static map-to-odom TF                     [0s]
  - YOLO object detection (GPU)              [10s]
  - Right-hand wall follower + sign react    [15s]

The robot will autonomously:
  - Follow the right wall to traverse the entire maze
  - Detect and react to signs:
      Stop_Sign  -> stop for 3 seconds, then resume
      Slow_Sign  -> reduce speed to 50%
      Fast_Sign  -> increase speed to 150%
  - Count detected objects (Orange, Tree, Car)
  - Save object counts to ~/ros2_ws/object_counts.txt


---------------------------------------------

## OPTIONAL: Individual run for debugging

# Step A: Start Simulation Environment
ros2 launch ntu_robotsim cwmaze.launch.py

# Step B: Spawn the Robot (Atlas)
ros2 launch ntu_robotsim single_robot_sim.launch.py

# Step C: Ground Truth Odometry to TF Broadcaster
ros2 launch odom_to_tf_ros2 atlas_odom_to_tf.launch.py

# Step D: OctoMap Server (3D Mapping)
ros2 launch octomap_server2 octomap_filtered.launch.py

# Step E: RViz2 Visualization
ros2 run rviz2 rviz2 -d $(ros2 pkg prefix ntu_robotsim)/share/ntu_robotsim/config/single_robot.rviz

# Step F: Nav2 Navigation Stack
ros2 launch nav2_bringup navigation_launch.py params_file:=$(ros2 pkg prefix ntu_robotsim)/share/ntu_robotsim/config/nav2_params.yaml use_sim_time:=true

# Step G: Static Map-to-Odom Transform
ros2 run tf2_ros static_transform_publisher 0 0 0 0 0 0 map atlas/odom

# Step H: Keyboard Teleop (Manual Control)
ros2 run teleop_twist_keyboard teleop_twist_keyboard --ros-args -r /cmd_vel:=/atlas/cmd_vel

# Step I: Wall Follower (standalone, if not using unified launch)
cd /home/ntu-user/ros2_ws && source install/setup.bash
python3 src/NTU_COMP30271_CW_RobotSim/ntu_robotsim/launch/wall_follower.py --ros-args -p use_sim_time:=true
